The short version

Four decisions shape everything below

One app, not five

Everyone downloads the same "HueVista" app. After sign-in it reads the account's role and shows that person's world. One app to maintain, one listing per store, one brand.

One codebase, both stores

Built with React Native + Expo: one TypeScript codebase that compiles into a real Android app and a real iPhone app. Your web frontend is already TypeScript + React — same skills, huge code reuse.

Your backend stays as-is

The app talks to the same Spring Boot REST API the website uses — same login, same catalogue, same projects, same billing. Zero backend rewrite to launch.

Same face as the website

Midnight Spectrum carries over: deep violet-black, electric purple #7c5cff, Space Grotesk headings. A retailer who knows the website recognises the app instantly.

How a mobile app actually gets made

You've never done mobile development, so here is the whole machine, honestly:

We write the app once, in TypeScript

React Native is a framework where you describe screens in code (very much like your Next.js site), and it produces native Android and iPhone interfaces — not a website in a wrapper. Expo is the toolkit around it that handles cameras, notifications, builds and updates.

Expo builds the installable files

One command produces an .aab file for Google Play and an .ipa file for the Apple App Store. During development you preview the app live on your own phone with the free "Expo Go" app — you'll see every screen running within days, not months.

The stores review and publish it

You'll need a Google Play developer account (₹2,100 one-time, approx.) and an Apple Developer account (₹8,300/year, approx.). Review takes a few hours on Google, a day or two on Apple. After that, updates ship the same way — and small fixes can even go out over-the-air without a store review.

The app talks to the server you already run

Every button in the app calls the same /api/… endpoints your website calls. Login, shade catalogue, projects, AI, Razorpay billing, access codes — all already built and tested. That is why this app is cheap to build: the hard part exists.

Screens · shared entry

The front door

First impressions do the selling. The welcome screen leads with the product — a live shade demo — and offers two honest paths: sign in, or "my paint shop gave me a code."

Screens · customer

The customer's world

The person repainting their home. Their app is built around one loop: photograph a room → try shades → save → share → order from their shop. Tabs: Home · Shades · Visualize · Projects · Account, with Visualize as the raised centre button.

Screens · retailer

The retailer's counter

The paying subscriber. Their app is a shop tool, designed for one hand while the other holds a tin of paint. Tabs: Counter · Codes · Orders · Painters · Account.

The walk-in flow, end to end: customer walks in → retailer taps New walk-in → photographs the customer's room (or receives it on WhatsApp) → walls auto-detected → they flip through shades together at the counter → retailer sends the result plus a 7-day code → the family decides at home in the app → the order comes back to this shop. The app's whole job is making that loop take under five minutes.

Screens · painter

The painter's day

Painters join through a retailer's invite link — the backend's painter-invitation flow. Their app answers three questions: where do I go, what exact shade and how much, and did I get paid. Tabs: Jobs · Visualize · Earnings · Account.

Screens · distributor

The distributor's territory

The distributor manages the retailer network your org-hierarchy module already models. Their app is a territory report they can read in a car between shop visits. Tabs: Network · Codes · Reports · Account.

Why an app at all

What the phone adds that the website can't

CapabilityWhat it means for HueVista
Camera in handThe room photo is the start of every sale. On mobile it's shoot → walls detected → shades flying, in under a minute — at the counter or on the customer's sofa.
Push notifications"Sunita opened your shared look 3 times" nudges the retailer to call. "New job assigned" reaches the painter on site. "Renewal in 3 days" reaches the retailer before the distributor has to chase. The website can't tap anyone on the shoulder.

The plan

How we build it, in order

Each phase ships something a real person uses. You'll be testing phase 1 on your own phone via Expo Go within the first week of building.

PHASE 1the product

Customer app: capture → visualize → share

The smallest app that sells paint. If only this ships, the app is already worth having.

  • Expo project scaffold, Midnight Spectrum theme, sign-in / register / Google (existing JWT endpoints)
  • Shop-code redeem as a first-class entry path
  • Camera + gallery capture → upload → your existing AI segmentation
  • The recolor engine on-device (GPU-drawn, same luminance-preserving approach as your WebGL) — instant, free color changes
  • Shade library (all 5 brands, cached offline) · projects · WhatsApp share
PHASE 2the business

Retailer counter mode

The subscriber's tool — this is what the ₹999/month is buying.

  • Counter dashboard: quota meter, walk-ins, orders
  • Two-tap access-code creation with WhatsApp hand-off
  • Walk-in flow tuned for speed (photo → shades → send, under 5 minutes)
  • Subscription status + Razorpay checkout in-app
  • Push notifications: shared-look opens, new orders, renewal reminders
PHASE 3the network

Painter & distributor worlds

  • Painter: invite redemption, job list & detail, completion photos, earnings
  • Distributor: network dashboard, renewal alerts, retailer invites
  • Storefront orders + wallet surfaces from your store/ module
PHASE 4the polish

Store launch & sharpening

  • App icons, splash screens, store listings (screenshots, descriptions in English + Kannada/Hindi)
  • Play Store + App Store submission and review shepherding
  • Analytics, crash reporting, over-the-air update channel for fast fixes

One honest caveat: the WebGL recolor engine is the only piece that doesn't copy across directly from the website — mobile uses the same GPU technique through a native drawing layer (Skia / GL). It's well-trodden ground, but it's the first thing I'd build in phase 1, so the riskiest part is proven earliest.

Recolor enginereact-native-skia / GLGPU-drawn, on-device, instant — keeps your "unlimited free color changes" economics intact on mobile.
PaymentsRazorpay checkout SDKOfficial React Native support; plugs into your existing subscriptions, plans and webhooks untouched.
Push notificationsExpo Notifications (FCM + APNs)One API for both platforms; a small backend addition stores device tokens and sends events (job assigned, look opened, renewal due).

Backend changesAlmost noneShip with the API as-is. Later niceties: device-token registration for push, a smaller image size for phone screens.
